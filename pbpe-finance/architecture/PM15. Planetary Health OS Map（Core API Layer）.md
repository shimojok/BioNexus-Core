ここからは、**BioNexus / Planetary Health OS の “全体マップ” を一度コンパクトに描き切ってから、実装フェーズに落としていきます。**

---

### 1. Planetary Health OS 全体マップ（コア API レイヤー）

**共通パターン（Stripe 型）：**

- **Intent:** 価値の「意図」・計算の中核  
- **Source:** データ源の標準化  
- **Webhook:** インパクト発生時のイベント（MRV 自動化）  
- **Connect:** 価値分配（40/40/20）  
- **Checkout:** 可視化 UI（誰にでも理解できる形）

**3つのドメイン：**

1. **AgriImpact（食料・土壌・生産問題）**  
   - Intent: `AgriImpactIntent`  
   - Source: AGRIX Soil/Crop/Water/MBT55  
   - Webhook: `soil.soc.updated` ほか  
   - Connect: 生産者 40 / PBPE基金 40 / PF 20  
   - Checkout: 圃場ごとの CO₂・土壌・収量・PBPE価値

2. **ClimateImpact（温室効果ガス・PBPE・気候ファイナンス）**  
   - Intent: `ClimateImpactIntent`  
   - Source: agriculture / energy / transport / waste  
   - Webhook: `co2.reduction.updated` ほか  
   - Connect: 行為主体 40 / PBPE基金 40 / PF 20  
   - Checkout: CO₂削減・固定・PBPE価値・ESG

3. **HealthImpact（グローバルヘルス・MBT・医療費）**  
   - Intent: `HealthImpactIntent`  
   - Source: HealthBook / MBT Probiotics / バイタル・生活習慣  
   - Webhook: `mbt55.updated` ほか  
   - Connect: ユーザー 40 / 予防基金 40 / PF 20  
   - Checkout: 健康改善・医療費削減・PBPE価値

**すべてが「価値創造 → 分配 → 再投資」の循環を前提に設計されている**ので、  
そのまま「資金循環エンジン」として機能します。

---

### 2. 資金循環のループ（あなたが本当にやりたいこと）

1. **現場でインパクトが発生する**  
   - 再生農業・省エネ・健康改善・コーヒー生産改善など

2. **Source → Intent に流れ込む**  
   - AGRIX / HealthBook / PBPE / MBT55 などのデータが標準化される

3. **Intent がインパクトと PBPE価値を計算する**  
   - CO₂・土壌・健康・収量・医療費削減 → PBPE価値

4. **Webhook が MRV と “価値の発生” を通知する**  
   - 「この圃場で 1.5t CO₂固定」「このユーザーで医療費 3万円削減」など

5. **Connect が 40/40/20 で価値を分配する**  
   - 現場（生産者・ユーザー）  
   - PBPE / 予防・気候基金  
   - プラットフォーム（持続性）

6. **Checkout が “見える化” して、次の行動を生む**  
   - 生産者・企業・自治体・投資家・消費者が「次の一手」を打ちやすくなる

→ これが **「価値創造と資金循環」の OS レベル実装**。

---

### 3. 次にやるべき「実装フェーズ」の優先順位

ここからは、もう概念ではなく「コードとエンドポイント」の世界です。  
優先度順に並べると：

1. **AgriImpactIntent v1 エンドポイント設計**  
   - `POST /agri/impact_intents`  
   - `GET /agri/impact_intents/{id}`  
   - MVP: SOC・収量・水・MBT55 から PBPE価値を返す

2. **AgriImpactWebhook v1（イベント定義＋シミュレーター）**  
   - `POST /webhooks/agri`  
   - soil / crop / water / mbt55 イベントの受信・検証

3. **AgriImpactConnect v1（40/40/20 分配ロジック）**  
   - Intent の `computed.pbpe_value_jpy` を受けて分配 JSON を返す

4. **AgriImpactCheckout v1（API → UI のデータ契約）**  
   - `GET /agri/impact_intents/{id}/checkout`  
   - そのままダッシュボードに載せられる形で返す

5. **Climate / Health モジュールは、Agri のパターンを継承して拡張**
