# 🧩 **🅓 PBPE Ledger — Data Model（完全版）**

PBPE Ledger は以下の 6 つのテーブル（またはコレクション）で構成されます。

```
PBPE Ledger
 ├── intents
 ├── pbpe_values
 ├── products
 ├── settlements
 ├── distributions
 └── audit_trail
```

Planetary Health OS の Intent Layer と PBPE Marketplace の金融商品が  
**この Ledger を通じて結びつきます。**

---

# 1. **Table: intents（Planetary Health OS からの入力）**

Intent Layer（Agri / Climate / Health）が生成する  
**価値の源泉データ**。

### **Schema**

```
intents {
  intent_id: string (PK)
  intent_type: enum("agri", "climate", "health")
  actor_id: string
  region: string
  timestamp_created: datetime
  timestamp_settled: datetime
  raw_data: object        // AGRIX / ClimateSource / HealthBook
  computed_metrics: object // SOC, CO2, MBT55, medical_cost, etc.
  pbpe_value: number
  status: enum("pending", "computed", "settled")
}
```

### **役割**

- PBPE Marketplace の **価値の源泉**
- MRV（Webhook Layer）と直結
- PBPE 計算の一次データ

---

# 2. **Table: pbpe_values（PBPE の一意性を保証）**

PBPE の **発行・追跡・二重計上防止** を行うレジストリ。

### **Schema**

```
pbpe_values {
  pbpe_id: string (PK)
  intent_id: string (FK)
  pbpe_amount: number
  pbpe_source: string   // soil, co2, mbt55, medical_cost, etc.
  timestamp_issued: datetime
  metadata: object
}
```

### **役割**

- PBPE の “発行台帳”
- カーボンクレジットの二重計上問題を完全に解決
- PBPE Marketplace の全金融商品の裏付け

---

# 3. **Table: products（PBPE 金融商品）**

PBPE Marketplace の金融商品（Bond / Credit / Insurance / Protection）。

### **Schema**

```
products {
  product_id: string (PK)
  product_type: enum("bond", "credit", "insurance", "protection")
  issuer_id: string
  underlying_intent: enum("agri", "climate", "health")
  pbpe_source: string
  expected_pbpe_yield: number
  risk_class: enum("A", "B", "C", "D")
  duration_months: integer
  payout_model: enum("fixed", "variable", "hybrid")
  distribution_model: string  // "40/40/20"
  currency: string
  region: string
  metadata: object
  status: enum("active", "matured", "settled")
}
```

### **役割**

- PBPE Marketplace の “商品カタログ”
- Intent と PBPE Ledger を金融商品に変換する

---

# 4. **Table: settlements（決済レコード）**

金融商品の満期・決済・PBPE 実現を記録する。

### **Schema**

```
settlements {
  settlement_id: string (PK)
  product_id: string (FK)
  pbpe_id: string (FK)
  pbpe_generated: number
  timestamp_settled: datetime
  settlement_method: string  // "intent_settled", "threshold_reached"
  ledger_hash: string        // tamper-proof hash
  metadata: object
}
```

### **役割**

- PBPE Marketplace の “決済台帳”
- Connect Layer（40/40/20）と連動
- 監査可能なトランザクション履歴

---

# 5. **Table: distributions（価値分配 40/40/20）**

Connect Layer が実行する **価値分配の詳細レコード**。

### **Schema**

```
distributions {
  distribution_id: string (PK)
  settlement_id: string (FK)
  actor_amount: number
  fund_amount: number
  platform_amount: number
  timestamp_distributed: datetime
  actor_wallet: string
  fund_wallet: string
  platform_wallet: string
  metadata: object
}
```

### **役割**

- 40/40/20 の透明性を保証
- PBPE Fund の蓄積を追跡
- 企業・政府・投資家向けの監査証跡

---

# 6. **Table: audit_trail（完全監査ログ）**

すべての Intent → PBPE → Settlement → Distribution の  
**完全な監査証跡（Audit Trail）**。

### **Schema**

```
audit_trail {
  audit_id: string (PK)
  entity_type: string   // intent, pbpe, product, settlement, distribution
  entity_id: string
  action: string        // created, updated, settled, distributed
  timestamp: datetime
  actor: string         // system, issuer, investor
  diff: object          // before/after changes
  hash: string          // cryptographic hash
}
```

### **役割**

- 国際監査基準（ISAE 3000）に対応
- カーボンクレジット市場の不正を根絶
- PBPE Marketplace の信頼性の基盤
