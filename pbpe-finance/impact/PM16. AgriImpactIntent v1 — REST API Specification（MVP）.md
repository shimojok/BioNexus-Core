# 🌾 **AgriImpactIntent v1 の “状態遷移”**

Stripe の PaymentIntent と同じ思想で設計。

```
pending → computed → settled
```

- **pending**：Intent 作成直後  
- **computed**：インパクト計算完了  
- **settled**：価値分配完了（Connect）  

この 3 ステージで、  
**価値創造 → 証明 → 分配 → 循環** が成立します。

---

# 🌍 **これで何ができるようになるか**

### 1. AGRIX → Intent → PBPE価値 → 分配  
圃場データがそのまま価値になる。

### 2. MRV（測定・報告・検証）が自動化  
Webhook と Intent の組み合わせで、  
CO₂・土壌・収量・水・MBT55 の変化が自動で証明される。

### 3. 生産者・PBPE基金・プラットフォームの資金循環が成立  
40/40/20 モデルが API レベルで実装される。

### 4. Checkout により “価値の見える化” が UX として完成  
企業・自治体・投資家が理解しやすい。
