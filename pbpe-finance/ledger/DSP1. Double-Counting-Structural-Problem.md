## 炭素クレジット・排出権取引における「二重計上」の構造的問題

従来のカーボンクレジット市場や排出権取引では、以下の構造的欠陥が存在します。

### ① 同一削減量の複数主張（ダブルクレーミング）
- 同じプロジェクト削減量を、異なる認証制度（VCS, Gold Standard, 国別レジストリ）が同時にクレジット発行
- 企業Aと企業Bが同じサプライヤーの削減量を「自社のScope 3削減」として報告

### ② 二重発行（ダブルイシュアンス）
- 同じトンCO₂に対して、複数のレジストリが独立してクレジットを発行
- 国際移転（ITMO）と国内削減が同一量をカウント

### ③ Scope 3 のカスケード二重計上
- 食品ロス削減：小売業者と食品メーカーが同じ削減量を主張
- 肥料削減：農家とバイヤーが同じ排出削減をScope 3で報告

### ④ レジストリ間の相互運用性欠如
- 各国・各制度ごとに独立した台帳 → グローバルな一意性保証が不可能

## PBPE Marketplace × ブロックチェーンによる優位点

PBPEは「**7桁コード（一意識別子）＋ PBPE Registry（集中台帳）＋ ブロックチェーン（改ざん不可）**」の三位一体構造で、従来問題を本質的に解決します。

### 🔹 優位点1：7桁コードによる「IDレベルでの衝突防止」
- 各PBPE Credit（`CRD-4F9A2C1`）、各Trade（`TRD-7C1A9F2`）、各Report（`RPT-2C9A1F7`）に**世界で唯一のID**
- 同じ削減量に2つの異なるIDが付与されることは物理的に不可能

### 🔹 優位点2：Intent → KPI → Credit の「因果チェーン」をコード化
- Impact Intent（`INT-xxxx`）→ KPI証明（`KPI-xxxx`）→ Credit発行（`CRD-xxxx`）
- 削減量の「発生源」が一意に特定され、二重主張を構造的に排除

### 🔹 優位点3：ブロックチェーンによる「改ざん不可能な単一台帳」
- PBPE Registryの全イベントをブロックチェーンに記録
- 一度書き込まれたCreditは「発行・移転・償却」の履歴が不変
- 同じCreditを2回売却・2回償却できない

### 🔹 優位点4：Scope 3 の「バリューチェーン全体」で一意性保証
- AGRIX（農場データ）→ MBT55（鮮度）→ HealthBook（栄養）→ PBPE Credit
- サプライチェーン上のどの企業も、「すでに誰かが主張した削減量」を再主張できない

### 🔹 優位点5：国連・ISOが求める「MRV基準」を完全充足
- Measurement：AGRIX等の実測データ
- Reporting：RPT-IDで一意報告
- Verification：ブロックチェーンハッシュによる自動監査

### 🌾 農業部門へのインパクト

| 指標 | 変化 |
|------|------|
| 土壌炭素 | +70〜200 tCO₂e/ha |
| 肥料使用量 | −20〜40% |
| 食品ロス | −15〜30% |
| 栄養価（Nutrition Index） | +10〜40% |
| 農家収入 | +25〜60%（PBPE Value Distribution） |

**構造的効果**：
- 農業が「収量産業」から「生態系価値産業」へ転換
- 各農地・各バッチに`INT-ID`が付与 → 全プロセスの価値化

### 🌍 気候変動ファイナンスへのインパクト

| 市場 | 年間規模（推定） |
|------|------------------|
| PBPE Credits（炭素・生防・食品ロス・品質） | 1.2〜2.5兆 USD |
| PBPE-backed Bonds | 0.3〜0.8兆 USD |
| PBPE Insurance | 0.1〜0.3兆 USD |

**構造的効果**：
- 二重計上が「物理的に不可能」な資産クラスとして成立
- PBPE Creditsが国際金融市場で取引可能な透明資産に
- Bond ID / Insurance IDによる追跡可能な気候金融

### 📊 ESG部門へのインパクト

| 課題 | PBPEによる解決 |
|------|----------------|
| データ不透明 | 全データに7桁コード＋ブロックチェーンハッシュ |
| 二重計上 | 同じ削減量を複数企業が主張不可 |

**構造的効果**：
- ESGが「コスト」から「収益源」に転換

- **二重計上ゼロ**の国際標準モデルとして、国連・世界銀行のMRV基準に適合
