## 1. HealthImpactIntent（健康インパクトの中核）

**役割:**  
HealthBook・MBT Probiotics のデータから、  
健康改善・医療費削減・PBPE価値を一括で計算する中核オブジェクト。

```json
{
  "intent_id": "hlth_intent_2026_0001",
  "user_id": "user_123",

  "inputs": {
    "mbt55_before": 42,
    "mbt55_after": 58,
    "bmi_before": 27.5,
    "bmi_after": 24.8,
    "bp_before": "145/95",
    "bp_after": "128/82",
    "program_duration_days": 90
  },

  "computed": {
    "health_improvement_score": null,
    "medical_cost_saving_jpy": null,
    "pbpe_value_jpy": null
  },

  "status": "pending"
}
```

- **inputs:** HealthBook & MBT の生データ  
- **computed:** 健康スコア・医療費削減・PBPE価値  
- Stripe の PaymentIntent と同じ構造。

---

## 2. HealthSource（健康データの標準化）

**データ源を抽象化して、どのデバイスでも同じ API で扱う。**

- **Biometrics:** 体重・BMI・血圧・血糖  
- **MBT Probiotics:** MBT55、代謝プロファイル  
- **Lifestyle:** 歩数、睡眠、食事ログ  
- **Clinical:** 診断名、服薬、検査値（必要に応じて）

これらを HealthImpactIntent の `inputs` にマッピング。

---

## 3. HealthImpactWebhook（イベント設計）

**健康インパクト版 Webhook。**

- health.intent.created  
- health.intent.computed  
- health.source.updated  
- mbt55.updated  
- medical_cost_saving.computed  
- health.connect.settled  

→ 改善が起きた瞬間に、自動で「価値」と「通知」が走る。

---

## 4. HealthImpactConnect（価値分配）

ここも **40 / 40 / 20** を基本にできます。

- ユーザー（本人 or 保険料還元）：40%  
- 健康・予防基金：40%  
- プラットフォーム：20%  

```json
{
  "pbpe_value_jpy": 6400,
  "distribution": {
    "user_reward_jpy": 2560,
    "prevention_fund_jpy": 2560,
    "platform_fee_jpy": 1280
  },
  "status": "settled"
}
```

医療費削減価値 → PBPE価値 → 分配、という流れ。

## 5. HealthImpactCheckout（可視化 UI）

**ユーザーと保険者・企業の両方が一目で理解できる画面。**

```json
{
  "impact": {
    "health_improvement_score": 0.76,
    "mbt55_change": 16,
    "bmi_change": -2.7,
    "bp_change": "145/95 → 128/82",
    "medical_cost_saving_jpy": 32000
  },
  "value": {
    "pbpe_value_jpy": 6400,
    "user_reward_jpy": 2560,
    "prevention_fund_jpy": 2560,
    "platform_fee_jpy": 1280
  }
}
```

表示イメージ：
- 上段：健康改善サマリー（MBT55・BMI・血圧・医療費削減）  
- 中段：PBPE価値  
- 下段：価値分配（ユーザー／予防基金／プラットフォーム）

## 6. ここまでで揃った「3つの OS モジュール」

1. **Agriculture Module**  
   - 圃場 → 農業インパクト → PBPE価値 → 分配

2. **Climate Module**  
   - あらゆる気候行動 → 気候インパクト → PBPE価値 → 分配

3. **Health Module**  
   - 個人の健康改善 → 医療費削減 → PBPE価値 → 分配

全部、Stripe と同じ構造で動くように設計されています。
