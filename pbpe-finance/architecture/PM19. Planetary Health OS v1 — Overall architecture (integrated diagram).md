# 🌱 1. Intent Layer（価値創造の心臓部）

Planetary Health OS の中心は **Intent** です。  
Stripe の PaymentIntent と同じ思想で、  
「価値の発生」を抽象化して扱えるようにする。

### Intent が担う役割
- データを受け取る  
- インパクトを計算する  
- PBPE価値を生成する  
- 状態遷移（pending → computed → settled）  
- Webhook を発火させる  
- Connect に価値を渡す  

### Intent の 3 本柱
- **AgriImpactIntent**（農業・土壌・食料）  
- **ClimateImpactIntent**（CO₂削減・気候価値）  
- **HealthImpactIntent**（健康改善・医療費削減）  

これらは **同じ構造で動く**ため、OS として統一される。

---

# 🌾 2. Source Layer（データ源の標準化）

Intent にデータを渡すための **標準フォーマット**。

### Agriculture
- SOC before/after  
- 収量 before/after  
- 水使用 before/after  
- MBT55 before/after  

### Climate
- CO₂削減量  
- CO₂固定量  
- エネルギー削減  
- 廃棄物削減  

### Health
- MBT55  
- BMI  
- 血圧  
- 歩数・睡眠  
- 医療費削減要因  

これらを **Source → Intent** に統一変換する。

---

# 🔔 3. Webhook Layer（MRV 自動化）

世界の課題の最大のボトルネックは **MRV（測定・報告・検証）**。  
あなたの OS はこれを **Webhook で完全自動化**する。

### 主なイベント
- soil.soc.updated  
- crop.yield.updated  
- water.use.updated  
- mbt55.updated  
- co2.reduction.updated  
- health.source.updated  
- intent.created  
- intent.computed  
- intent.settled  

これにより、  
**インパクトが発生した瞬間に価値が証明される**。

---

# 💸 4. Connect Layer（価値分配）

あなたが定義した **40 / 40 / 20 モデル** を OS に組み込む。

### 分配モデル
- 40%：現場（生産者・企業・ユーザー）  
- 40%：PBPE / 予防・気候基金  
- 20%：プラットフォーム（BioNexus）  

Stripe Connect の「transfer」を、  
**地球価値の分配**に置き換えたもの。
